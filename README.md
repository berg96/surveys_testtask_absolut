# Survey System

## Описание проекта

Мини-сервис для управления пользовательскими опросами.  
Пользователи могут создавать опросы, редактировать их и собирать статистику (среднее время прохождения, количество начатых и завершенных прохождений).  
Другие пользователи могут проходить эти опросы, отвечать на вопросы и получать следующий вопрос до завершения опроса.  

Опрос состоит из:
* Названия
* Автора
* Даты создания  
Вопросы имеют:
* Варианты ответов
* Возможность "своего ответа"
* Порядок отображения, задаваемый автором  

---

## Реализованные эндпоинты

| Метод | Путь                                     | Описание                                                        |
|-------|------------------------------------------|-----------------------------------------------------------------|
| GET   | `/api/surveys/{pk}/next-question/`       | Получение следующего вопроса для прохождения опроса             |
| GET   | `/api/surveys/{pk}/stats/`               | Получение статистики опроса (только автор)                      |

---

## Статистика опроса

* Подсчитывается общее количество сессий опроса
* Количество завершённых сессий
* Среднее время прохождения (в секундах)  
* Доступно только автору опроса

## Как запустить проект

1. Клонировать репозиторий:
```
git clone https://github.com/berg96/surveys_testtask_absolut
```
2. Перейти в папку проекта:
```
cd surveys_testtask_absolut
```
3. Создать файл .env на основе примера .env.example и заполнить переменные окружения:

4. Запустить все сервисы через Docker Compose:
```
docker compose up -d
```

Документация API при запущенных контейнерах доступна по следующим адресам:
[Swagger](http://localhost/api/docs/swagger)
[Redoc](http://localhost/api/docs/redoc)
